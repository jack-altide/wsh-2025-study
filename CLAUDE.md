# WSH 2025 学習プロジェクト

## プロジェクト概要

Web Speed Hackathon 2025 のコードを理解するための学習タスク管理リポジトリです。

## 技術スタック

### フロントエンド
- TypeScript / React 19 / React Router 7
- Zustand（状態管理）
- UnoCSS（スタイリング）
- HLS.js / Video.js（動画再生）

### バックエンド
- Node.js / Fastify 5
- SQLite + Drizzle ORM
- SSR（サーバーサイドレンダリング）

### 開発ツール
- pnpm（パッケージ管理）
- Webpack 5（バンドル）
- Playwright（テスト）

## 学習フェーズ

| Phase | テーマ |
|-------|--------|
| 1 | 言語とツールの基礎（TypeScript、pnpm） |
| 2 | フロントエンド基礎（React、状態管理、スタイリング） |
| 3 | バックエンド基礎（Fastify、ORM、API設計） |
| 4 | アーキテクチャ理解（SSR、バンドル、最適化） |
| 5 | 実践（パフォーマンス改善） |

## 利用可能なコマンド

- `/learn` - 新しい学習トピックを開始する
- `/review` - 学習内容を復習する
- `/progress` - 学習進捗を確認する
- `/explain` - 技術概念を詳しく説明する

## ルール

### 学習サポート時のガイドライン

1. **段階的な説明**: 複雑な概念は基礎から段階的に説明する
2. **実例重視**: 抽象的な説明より具体的なコード例を示す
3. **関連付け**: 新しい概念は既知の概念と関連付けて説明する
4. **確認質問**: 理解度を確認するための質問を適宜行う

### ノート作成時のルール

1. 学習ノートは `notes/phaseX/` に配置する
2. ファイル名は `topic-name.md` 形式にする
3. 各ノートには以下を含める:
   - 概要
   - 重要なポイント
   - コード例
   - 参考リンク

### 練習問題のルール

1. 練習コードは `exercises/` に配置する
2. 各練習問題には解説コメントを含める
